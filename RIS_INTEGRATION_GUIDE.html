⚙️ Step 1: Database Setup

Execute Database Migration

Copy and execute the SQL migration to your PostgreSQL database.

📁 server/database/migration_ris_transactions_v3.sql

Execution Methods:

  • Using psql command line:
psql -U postgres -d procurement_db -f server/database/migration_ris_transactions_v3.sql
  • Using pgAdmin GUI: Open Tools → Query Tool → Copy-Paste the SQL → Execute
  • Using Node.js (automatic): Add to server startup script
⚠️ Important: Ensure the following base tables exist before running migration:
  • users table
  • employees table
  • items table (master catalog)
  • requisition_issue_slips table
  • designations table

What Gets Created:

  • Additional columns in requisition_issue_slips table (5 new columns)
  • Additional columns in ris_items table (2 new columns)
  • Additional columns in items table (2 new columns)
  • ris_notifications table (for role-based visibility)
  • ris_approvals table (for approval tracking)
  • ris_status_history table (for audit trail)
  • Database view: vw_ris_complete
  • Database view: vw_ris_items_with_stock
  • Stored procedure: add_manual_item_to_catalog()
  • Stored procedure: create_ris_with_notifications()
  • Stored procedure: approve_ris()
✅ Verify Success: Check if new tables exist in your database:
SELECT * FROM ris_notifications LIMIT 0; SELECT * FROM ris_approvals LIMIT 0; SELECT * FROM ris_status_history LIMIT 0;

// Look for: // GET /api/ris - List all RIS app.get('/api/ris', authenticateToken, async (req, res) => {

Replace the entire RIS section with the content from ris-endpoints-v3.js.

Required Dependencies Check:

// Make sure these are available in server.js: const pool = require('../database'); // PostgreSQL pool const authenticateToken = require('./middleware/auth'); // JWT middleware // Ensure this function exists: function broadcastNotification(event, data) { if (global.io) { global.io.emit(event, data); } }
⚠️ Important: The endpoints use these roles: HOPE, BAC_SEC, BUDGET, CHIEF_FAD, ADMIN, SUPPLY_OFFICER. Ensure your user system supports these roles.

🔌 Step 7: Socket.IO Real-Time Notifications (Optional)

Enable Real-Time Notifications

The API endpoints already include Socket.IO broadcasts. To enable real-time notifications:

1. Verify Socket.IO is initialized in server.js:

const io = require('socket.io')(server, { cors: { origin: "*" } }); global.io = io; // Make available to all modules

2. Add client-side listener in your main app.js:

// Connect to Socket.IO if (typeof io !== 'undefined') { const socket = io(); // Listen for RIS notifications socket.on('ris-created', function(data) { console.log('New RIS created:', data); showToast('New RIS ' + data.ris_no + ' created!', 'info'); loadRISList(); }); socket.on('ris-approved', function(data) { console.log('RIS approved:', data); showToast('RIS ' + data.ris_no + ' approved!', 'success'); loadRISList(); }); }

3. Notifications are sent for:

  • New RIS creation (to all stakeholder roles)
  • RIS approval (to requestor and all roles)
  • RIS cancellation (to all involved parties)
  • Manual item added to catalog (to SUPPLY_OFFICER role)
ℹ️ Note: Even without Socket.IO, notifications are stored in ris_notifications table and users will see pending RIS when they call GET /api/ris/notifications/pending

📞 Support & Troubleshooting

Common Issues & Solutions

Error: "Cannot find module './ris-endpoints-v3'"

Solution: Ensure file path is correct and file exists in server/ directory. Use relative or absolute path in require statement.

Error: "RIS_NOTIFICATIONS table does not exist"

Solution: Run the SQL migration file. Check that all SQL executed without errors.

Issue: UI modals not appearing

Solution:
1. Check CSS is loaded: F12 → Styles tab → search for "ris-modal"
2. Check HTML is in page: F12 → Elements tab → search for "risModal"
3. Check JS functions: Open browser console, type typeof showNewRISForm should return "function"

Issue: Buttons not working / 404 errors

Solution:
1. Check server is running: curl http://localhost:3000/
2. Check API endpoints integrated: Search server.js for "app.get('/api/ris"
3. Check authentication token: localStorage.getItem('token') in console

Issue: Database errors on RIS creation

Solution:
1. Check user exists: SELECT * FROM users WHERE id = X;
2. Check items exist: SELECT * FROM items LIMIT 5;
3. Check items table has correct structure: \d items in psql
